## 1. 工作背景与研究动机

炼厂多装置操作调度属于典型的流程工业协同调度问题。与离散制造调度不同，炼厂调度具有以下特点：

1. **装置链条长、物料耦合强**  
   原油经过常减压、催化裂化、加氢、调和等多级装置后形成汽油、柴油、LPG 等产品。上游装置负荷会影响下游进料，中间缓冲罐和产品池又会反过来约束装置运行。

2. **库存和物流约束显著**  
   调度不仅要决定各装置处理负荷，还要保证中间库存、产品库存、装置能力和需求约束可行。

3. **运行扰动频繁**  
   实际生产中需求、产品价格、原油成本、装置可用率和收率都存在不确定性，固定优化方案在扰动下可能表现下降。

4. **传统优化与强化学习各有局限**  
   - MILP/MINLP 能精确表达约束，但在线重调度和复杂不确定场景下求解成本较高；
   - 纯 RL/MARL 具有自适应能力，但如果缺少领域约束，容易产生不可执行或低质量动作。

因此，本工作希望探索一种 **领域知识约束增强的安全多智能体强化学习方法**，在保证可行性的同时提升不确定环境下的自适应调度性能。

## 2. 问题定义与建模

### 2.1 调度对象

当前案例为 24 个调度时段的炼厂多装置操作调度问题，主要装置包括：

| 智能体/装置 | 说明 |
|---|---|
| CDU1 | 常减压装置 1 |
| CDU2 | 常减压装置 2 |
| DFHC | 柴油加氢精制装置 |
| FCC1 | 催化裂化装置 1 |
| FCC2 | 催化裂化装置 2 |
| ROHU | 渣油加氢装置 |
| DHC | 柴油加氢裂化装置 |

库存节点包括原油罐、中间缓冲罐、调和组分池和最终产品池。产品包括汽油池、柴油池和 LPG 池对应的多个产品牌号。

### 2.2 确定性数学规划基线

项目中已实现 Gurobi 全周期数学规划模型：

文件：`refinery_gurobi_model.py`

当前模型属于 **full-horizon deterministic MILP/MINLP baseline**，不是 rolling MILP/MINLP。其特点是一次性优化完整 24 个时段，决策变量覆盖全周期。

主要变量包括：

- 原油采购量 `crude_buy[c,t]`
- 原油库存 `crude_inv[c,t]`
- CDU 原油进料 `crude_feed[c,u,t]`
- 装置处理量 `unit_charge[u,t]`
- 装置开停状态 `unit_on[u,t]`
- 装置模式变量 `mode_on[u,m,t]`
- 中间库存 `inventory[node,t]`
- 产品池库存 `product_inventory[pool,t]`
- 产品销售量 `product_sale[p,t]`

目标函数为最大化全周期利润：

```text
max product_revenue
    - crude_purchase_cost
    - utility_cost
    - inventory_holding_cost
    - switching_cost
    - load_stability_cost
```

| revenue | 3,950,519.75 |
| crude cost | 2,197,415.46 |
| utility cost | 23,122.42 |
| inventory cost | 144,218.90 |
| switching cost | 18,800.00 |
| load stability cost | 1,208.22 |

### 2.3 多智能体强化学习建模

将调度问题建模为多智能体马尔可夫决策过程：

```text
<I, S, {O_i}, {A_i}, P, R>
```

其中：

- `I`：装置智能体集合，共 7 个；
- `S`：全局状态，包括装置负荷、库存、产品池、需求、不确定扰动等；
- `O_i`：智能体局部观测，包括自身负荷、上下游库存、相关需求、邻近装置状态等；
- `A_i`：离散负荷动作；
- `P`：由物料路由、库存更新、调和和不确定扰动共同决定；
- `R`：利润导向奖励，包含收入、原油成本、公用工程成本、库存成本、短缺惩罚和违规惩罚。

## 3. 所提方法：KC-Safe-MAPPO

### 3.1 方法名称

**KC-Safe-MAPPO**  
Knowledge-Constrained Safe Multi-Agent PPO

中文可表述为：

**领域知识约束增强的安全多智能体 PPO 调度方法**

### 3.2 方法框架

方法采用集中训练、分散执行（CTDE）框架：

```text
案例数据 config.py
      ↓
ConfigAdapter 数据适配
      ↓
RefinerySchedulingEnv 炼厂调度环境
      ↓
各装置 Actor 输出候选负荷动作
      ↓
SafetyLayer 基于领域知识进行动作掩码与修正
      ↓
Routing / Blending 计算物料流、库存和产品交付
      ↓
RewardCalculator 计算利润导向奖励
      ↓
Centralized Critic + Shared Actor 进行 MAPPO 更新
```

### 3.3 领域知识注入方式

目前知识约束主要体现在四个层面：

1. **动作可行域约束**
   - 装置能力上下限；
   - 上游物料不足时禁止高负荷动作；
   - 下游库存接近上限时限制继续生产；
   - 装置可用率降低时动态降低可行动作上限。

2. **物料平衡与库存动态**
   - CDU 产物进入中间缓冲罐或直接进入下游装置；
   - 二次加工装置按收率产生调和组分；
   - 组分池进入产品池；
   - 产品池满足需求并产生销售收入。

3. **经济目标对齐**
   - 收入按产品牌号价格计算，而不是简单产品池平均价；
   - 原油成本、库存成本、公用工程成本与 Gurobi 目标函数口径保持一致；
   - demand max 中的 `BIG_M` 被视为无穷上限，避免错误限制销售。

4. **不确定场景下的 profit guard**
   - 当原油价格相对产品价格偏高时，CDU 负荷从满负荷下调一档；
   - 二次加工装置保持较高负荷，以维持产品转化能力；
   - 该规则作为知识驱动的 action improvement 层，用于提升不确定环境下的鲁棒收益。

### 3.4 MAPPO 训练改进

当前已加入以下训练稳定性增强：

- advantage normalization；
- value clipping；
- centralized critic minibatch update；
- actor / critic orthogonal initialization；
- LayerNorm 提升网络训练稳定性；
- best checkpoint 保存机制；
- entropy annealing；
- reward fusion 中提高全局利润权重，目前 `alpha = 0.95`。

## 4. 不确定性建模与对比方法

### 4.1 不确定因素

在原确定性环境基础上，引入中等扰动场景（moderate uncertainty），包括：

| 不确定因素 | 建模方式 |
|---|---|
| 产品需求 | demand multiplier |
| 产品价格 | price multiplier |
| 装置收率 | yield multiplier |
| 原油成本 | crude price multiplier |
| 装置可用率 | unit availability / derating |

当前 moderate profile 的主要设置：

| 参数 | 数值 |
|---|---:|
| demand std | 0.12 |
| price std | 0.08 |
| yield std | 0.05 |
| crude price std | 0.07 |
| unit outage probability | 0.03 |
| unit derate minimum | 0.65 |

### 4.2 对比方法

目前已实现以下不确定环境对比方法：

1. **KC-Safe-MAPPO**
   - 在 moderate 不确定环境中重新训练；
   - 训练后使用 best actor；
   - 评估时使用 profit guard 动作改进层。

2. **随机规划 SAA baseline**
   - 基于多个训练场景采样；
   - 在候选分段负荷计划中选择平均利润最高的方案；
   - 可理解为 scenario average approximation 的离散近似版本。

3. **鲁棒优化 CVaR baseline**
   - 在 stress profile 下采样训练场景；
   - 以场景尾部平均收益作为目标；
   - 选择尾部表现较好的固定分段计划。

4. **随机策略 baseline**
   - 每个时步在安全可行动作中随机选择；
   - 用于验证方法相对随机动作的有效性。

说明：当前随机规划和鲁棒优化是 **scenario-based discrete planning baseline**，不是完整 stochastic MILP 或 robust counterpart。后续可进一步扩展为 Gurobi-based stochastic MILP / robust MILP。

## 6. 实验结果

### 6.1 不确定环境对比结果

moderate 不确定环境下，24 个测试场景的对比结果如下：

| 方法 | mean profit | std profit | min profit | max profit |
|---|---:|---:|---:|---:|
| KC-Safe-MAPPO + profit guard | 1,016,655.54 | 100,680.02 | 853,398.92 | 1,242,025.38 |
| 随机规划 SAA | 1,004,761.24 | 104,414.25 | 839,659.75 | 1,232,837.55 |
| 鲁棒优化 CVaR | 1,004,761.24 | 104,414.25 | 839,659.75 | 1,232,837.55 |
| 随机策略 | 301,981.94 | 147,219.06 | 2,199.42 | 518,700.79 |

### 6.2 相对提升

KC-Safe-MAPPO 相比 SAA / CVaR baseline：

| 指标 | 提升 |
|---|---:|
| mean profit | +11,894.31 |
| min profit | +13,739.18 |
| max profit | +9,187.84 |
| std profit | -3,734.23 |

说明：

- KC-Safe-MAPPO 在平均收益上超过随机规划和鲁棒优化；
- 在最差场景收益上也更高，说明不仅提高均值，也改善了尾部表现；
- 标准差略低，说明收益波动性略有下降；
- 随机规划和鲁棒优化在当前候选空间中选到了相同计划，说明粗粒度动作空间下均值最优和尾部最优方案一致。

### 6.3 主要观察

1. **确定性高负荷计划是很强的 baseline**  
   随机规划和鲁棒优化当前都倾向于选择全装置高负荷计划。这说明在案例数据下，高负荷生产通常仍具有较高经济性。

2. **纯 MAPPO 容易偏保守**  
   初始 MARL 策略会受到局部能耗和切换惩罚影响，出现 revenue 偏低的问题。

3. **领域知识 action improvement 有明显作用**  
   profit guard 利用“原油相对产品价格”判断 CDU 是否需要降档，解决了固定高负荷计划在高原油成本场景下的不足。

4. **不确定状态观测是必要的**  
   如果 actor 看不到价格、需求、收率、原油成本等扰动，只能学习平均策略，难以体现自适应优势。

## 8. 阶段性结论

目前工作已完成从确定性调度到不确定调度评估的初步闭环：

1. 构建了炼厂多装置调度的 Gurobi full-horizon MILP/MINLP 基线；
2. 实现了 KC-Safe-MAPPO 多智能体强化学习方法；
3. 完成了环境与 Gurobi 经济口径对齐，包括产品牌号收入、需求上限和库存成本；
4. 引入需求、价格、收率、原油成本和装置可用率不确定性；
5. 实现了随机规划 SAA 和鲁棒优化 CVaR 的场景采样对比；
6. 通过不确定状态观测、全局利润奖励强化和 profit guard，使 KC-Safe-MAPPO 在 moderate 不确定场景下超过 SAA / CVaR baseline。

当前最重要的阶段性结果是：

> 在 24 个 moderate 不确定测试场景下，KC-Safe-MAPPO + profit guard 的 mean profit 为 1,016,655.54，高于随机规划和鲁棒优化的 1,004,761.24；同时 min profit 也从 839,659.75 提升到 853,398.92。

## 9. 目前不足与需要进一步讨论的问题

### 9.1 方法层面

1. **profit guard 目前仍是规则增强层**  
   它提升效果明显，但需要进一步讨论其定位：
   - 是作为 KC-Safe-MAPPO 的知识约束执行层；
   - 还是作为后处理 action improvement；
   - 论文中需要表述清楚，避免被理解为单纯启发式替代 RL。

2. **SAA / CVaR baseline 仍是离散候选近似**
   当前随机规划和鲁棒优化尚不是完整 stochastic MILP / robust MILP。后续若要增强说服力，可实现：
   - scenario-based two-stage stochastic MILP；
   - box uncertainty robust counterpart；
   - rolling-horizon stochastic / robust MILP。

3. **MARL 策略仍有训练波动**
   不同 seed 下表现存在差异，需要：
   - 多随机种子重复训练；
   - 报告均值和置信区间；
   - 保存并评估 best checkpoint。

### 9.2 建模层面

1. 当前产品质量约束尚未完整进入 MARL 环境；
2. 调和模型仍是聚合产品池调和，未细化到具体组分质量；
3. 装置模式对收率和能耗的影响仍较简化；
4. 不确定性分布目前采用截断正态扰动，后续可根据真实数据估计分布。
